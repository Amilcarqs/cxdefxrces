# Reach Value — Explicación de la Recursión

El problema pregunta:

> “¿Es posible llegar exactamente a `N`
> comenzando desde `1`
> usando únicamente estas operaciones?”

```text
×10
×20
```

---

# Idea principal de la recursión

La recursión explora todas las posibilidades.

Desde un valor actual `x`, solo existen dos movimientos posibles:

```text
x * 10
x * 20
```

Entonces la recursión piensa así:

> “Si puedo llegar a `N`
> desde `x*10`
> o desde `x*20`,
> entonces también puedo llegar desde `x`.”

---

# Relación recursiva

La función realmente significa:

```text
f(x) = ¿puedo llegar a N comenzando desde x?
```

y explora:

```text
f(x)=f(x*10) OR f(x*20)
```

---

# ¿Por qué funciona?

Porque cada llamada recursiva representa:

> “Resolver el mismo problema,
> pero comenzando desde otro valor.”

La recursión reutiliza exactamente la misma lógica una y otra vez.

---

# Casos base

Los casos base son las condiciones que detienen la recursión.

Sin ellos, las llamadas continuarían infinitamente.

---

## Caso base 1: llegaste exactamente a N

```text
x == N
```

Significa:

> “Encontré un camino válido.”

Entonces retorna:

```text
1
```

o verdadero (`true`).

---

## Caso base 2: te pasaste de N

```text
x > N
```

Significa:

> “Este camino ya no sirve.”

Entonces retorna:

```text
0
```

o falso (`false`).

---

# Cómo piensa la recursión

Desde cada valor se generan dos nuevas posibilidades:

```text
valor actual
├── multiplicar por 10
└── multiplicar por 20
```

Cada operación genera un nuevo subproblema.

---

# Árbol recursivo

Ejemplo conceptual:

```text
f(1)
├── f(10)
│   ├── f(100)
│   │   ├── f(1000)
│   │   └── f(2000)
│   └── f(200)
│       ├── f(2000)
│       └── f(4000)
└── f(20)
    ├── f(200)
    └── f(400)
```

La recursión explora todas las ramas posibles buscando llegar exactamente a `N`.

---

# Cómo se obtiene la respuesta

Cada rama puede terminar de dos formas:

```text
llegué exacto → válido
me pasé       → inválido
```

Si al menos una rama llega exactamente al objetivo:

```text
YES
```

Si todas fallan:

```text
NO
```

---

# Idea importante

Este problema NO busca contar caminos.

La pregunta es:

```text
¿Existe al menos una forma válida?
```

Por eso basta con que una sola rama funcione.

---

# Diferencia con problemas de conteo

En problemas de conteo:

```text
respuesta = suma de caminos
```

Aquí:

```text
respuesta = existe o no existe
```

Entonces:

```text
camino válido → YES
ningún camino → NO
```

---

# Patrón general de este tipo de recursión

Muchos problemas siguen esta estructura:

```text
resolver(estado actual):

    si llegué exactamente:
        retornar verdadero

    si me pasé:
        retornar falso

    probar todas las decisiones posibles
```

---

# Idea mental correcta

Debes pensar:

> “Desde mi estado actual,
> pruebo todas las operaciones posibles.
>
> Si alguna llega al objetivo,
> entonces la respuesta es YES.”
