# 🧠 Problema: 0/1 Knapsack (Mochila)

## 📌 Idea general

Tienes:

- N objetos
- Cada objeto tiene:
  - un **peso**
  - un **valor**
- Una mochila con capacidad máxima **W**

Tu objetivo es:

> Elegir un subconjunto de objetos (sin repetir ninguno) para maximizar el valor total sin exceder el peso W.

---

# ⚙️ Tipo de problema

Este es un problema clásico de:

- **Backtracking / Recursión**
- **Programación dinámica (DP)**
- Decisión binaria por elemento

Cada objeto tiene solo dos opciones:

- Tomarlo
- No tomarlo

---

# 🌳 Modelo mental (recursión)

El problema se puede ver como un árbol de decisiones:

Para cada objeto `i`:

- Rama 1: no tomarlo
- Rama 2: tomarlo (si entra en la mochila)

Esto genera todas las combinaciones posibles.

---

# 🧩 Estado de la recursión

La función se define como:

> “Máximo valor posible usando los objetos desde i hasta el final, con una capacidad restante W”

El estado depende de:

- `i` → qué objeto estás evaluando
- `W` → capacidad restante

---

# 🔁 Transiciones

Para cada objeto tienes dos opciones:

## 1. No tomar el objeto

- No cambia el peso
- Pasas al siguiente objeto

## 2. Tomarlo (solo si entra)

- Sumas su valor
- Restas su peso a la capacidad
- Pasas al siguiente objeto

---

# 🎯 Regla principal

Siempre eliges:

> el máximo entre tomar y no tomar

---

# 🚪 Casos base

La recursión termina cuando:

## 1. Ya no quedan objetos

- Si `i == N`
- No hay más decisiones posibles
- Resultado = 0

## 2. (implícito) capacidad agotada

- Si W llega a 0, ya no puedes agregar más objetos

---

# ⚠️ Errores comunes

## ❌ Confundir peso con valor

- El peso se usa para la capacidad
- El valor se suma como ganancia

---

## ❌ Pensar que un objeto que no entra termina el problema

Incorrecto:

- Si un objeto no entra, solo se ignora
- No detiene el proceso

---

## ❌ Caso base incorrecto

El error más común es detener la recursión en el último objeto en lugar de cuando se han evaluado todos.

---

# 📊 Complejidad

## Sin optimización (recursión pura)

- Cada objeto tiene 2 decisiones

→ Complejidad:

O(2^N)

---

## Con límites del problema

- N ≤ 20

Esto es suficiente para que la recursión funcione.

---

# 🚀 Optimización (idea futura)

Este problema tiene subproblemas repetidos:

Ejemplo:

- (i = 5, W = 10) puede aparecer muchas veces

Esto permite usar:

- memoización
- programación dinámica

Reduciendo complejidad a:

O(N × W)

---

# 🧠 Idea clave para recordar

> Cada objeto es una decisión independiente: tomarlo o ignorarlo.

Y el problema es:

> explorar todas las combinaciones válidas y quedarte con la mejor.

---

# 🔥 Resumen mental

- Estado: (i, W)
- Decisión: tomar / no tomar
- Respuesta: max de ambas opciones
- Fin: cuando no quedan objetos
