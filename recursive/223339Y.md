# Recursión en problemas de conteo de caminos

Este tipo de problemas pregunta:

> “¿Cuántas formas distintas existen para llegar al destino?”

La idea de la recursión es dividir el problema grande en problemas más pequeños.

---

# Idea principal de la recursión

Supongamos que estás en una posición `x`.

Desde ahí puedes hacer varios movimientos:

```text
+1
+2
+3
```

Entonces la pregunta cambia a:

> “¿Cuántas formas existen si avanzo 1?”
>
> “¿Cuántas formas existen si avanzo 2?”
>
> “¿Cuántas formas existen si avanzo 3?”

La respuesta total será la suma de todas esas posibilidades.

---

# Relación recursiva

Matemáticamente:

```text
f(x)=f(x+1)+f(x+2)+f(x+3)
```

donde:

```text
f(x) = cantidad de formas de llegar desde x hasta E
```

---

# ¿Por qué funciona?

Porque cada llamada recursiva representa:

> “Resolver el mismo problema, pero comenzando desde otra posición.”

La recursión reutiliza exactamente la misma lógica una y otra vez.

---

# Casos base

Los casos base son las condiciones que detienen la recursión.

Sin ellos, las llamadas continuarían infinitamente.

---

## Caso base 1: llegaste exactamente al destino

```text
actual == E
```

Significa:

> “Encontré un camino válido.”

Entonces retorna:

```text
1
```

porque existe exactamente una forma válida.

---

## Caso base 2: te pasaste del destino

```text
actual > E
```

Significa:

> “Este camino ya no sirve.”

Entonces retorna:

```text
0
```

porque no existe ninguna forma válida desde allí.

---

# Cómo piensa la recursión

La recursión explora todas las decisiones posibles.

Desde cada posición:

```text
posición actual
├── mover +1
├── mover +2
└── mover +3
```

Cada movimiento genera nuevos subproblemas.

---

# Árbol recursivo

La recursión puede visualizarse como un árbol.

Ejemplo:

```text
f(2)
├── f(3)
│   ├── f(4)
│   │   ├── f(5) = 1
│   │   ├── f(6) = 0
│   │   └── f(7) = 0
│   ├── f(5) = 1
│   └── f(6) = 0
├── f(4)
│   ├── f(5) = 1
│   ├── f(6) = 0
│   └── f(7) = 0
├── f(5) = 1
```

---

# Cómo se obtiene la respuesta

Cada rama devuelve:

- `1` si llega exactamente al destino
- `0` si se pasa

Luego todas las respuestas se suman.

```text
1 + 1 + 1 + 1 = 4
```

---

# Interpretación importante

La recursión no “adivina” la respuesta.

Lo que hace es:

1. Probar todos los movimientos posibles
2. Resolver recursivamente cada posibilidad
3. Contar cuáles terminan correctamente
4. Sumar todas las respuestas válidas

---

# Patrón general en problemas similares

Muchos problemas de recursión siguen esta estructura:

```text
resolver(estado actual):

    si llegué exactamente:
        retornar 1

    si el estado ya no sirve:
        retornar 0

    retornar suma de todas las decisiones posibles
```

---

# Idea mental correcta

Debes pensar la recursión así:

> “La respuesta desde una posición depende de las respuestas de posiciones más pequeñas o más cercanas al final.”

La recursión divide continuamente el problema hasta llegar a casos simples que sí puede responder directamente.
